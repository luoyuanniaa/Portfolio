# Design Spec — ID Portfolio
参考基准:**takram.com**(克制、编辑感、留白驱动)
版本:v1 · 待你确认后落地到 Figma 框架与网页

---

## 0. 一句话方向
像一本安静的作品集杂志:**大量留白、强排版层级、方正的大图**。
靠空白和字号区分内容,**不靠框、不靠线**。

---

## 1. 设计原则(硬性约束)

| 原则 | 说明 |
|---|---|
| **零圆角** | 所有元素 `border-radius: 0`。图片、色块、按钮、输入框一律方正。 |
| **无框无线** | 不用文本框、卡片描边、分割线来切分内容。区块之间靠**留白**分隔。 |
| **无阴影** | 默认不用投影。层级靠字号、字重、颜色与间距表达。 |
| **内容优先** | 图与字是主角,UI 装饰降到最低。导航、页脚都极简。 |
| **单色为主** | 近乎黑白灰;颜色克制,必要时只用一个低调强调色(默认不上色)。 |
| **大图方正** | 项目图满幅或大尺寸、方角呈现,不加边框、不加圆角。 |

> 例外:**极少数**情况若必须分隔,只允许一条 8% 透明度的 1px 细线(hairline),且仅在确有必要时使用。默认一条都不用。

## 2. 版式与网格 Layout & Grid

| Token | 值 |
|---|---|
| 最大内容宽 `--maxw` | 1440px(超宽屏留白居中) |
| 外边距 `--margin` | `clamp(24px, 6vw, 96px)` |
| 栅格 | 12 列,gutter `clamp(20px, 2vw, 32px)` |
| 区块竖向留白(桌面) | `clamp(120px, 16vh, 200px)` |
| 区块竖向留白(移动) | `clamp(64px, 10vh, 96px)` |
| 间距阶梯 | 4 · 8 · 12 · 16 · 24 · 32 · 48 · 64 · 96 · 128 · 160 · 200 |

要点:**留白要够大、够敢**。项目与项目之间用大段空白拉开,而不是画线。

## 3. 排版 Typography

字体追求 Takram 那种**中性 grotesque + 紧字距**的编辑感。

| 用途 | 字号(桌面) | 字重 | 字距 | 行高 |
|---|---|---|---|---|
| Hero / Display | `clamp(56px, 8vw, 96px)` | 400–500 | -2% | 1.0–1.05 |
| H2 区块标题 | `clamp(36px, 5vw, 56px)` | 400 | -1.5% | 1.08 |
| H3 项目标题 | `clamp(28px, 3.4vw, 44px)` | 500 | -1% | 1.05 |
| Lead 引言 | `clamp(19px, 2vw, 24px)` | 300–400 | 0 | 1.4 |
| Body 正文 | 16–17px | 400 | 0 | 1.6 |
| Label 标签/类目 | 12–13px | 500 | +6% | 1.2 |
| Meta 信息行 | 12–13px | 400 | +2% | 1.4 |

**字体族建议**
- 拉丁(首选,授权):`Founders Grotesk` / `Neue Haas Grotesk` / `ABC Diatype` 任一(最接近 Takram 气质)。
- 拉丁(免费可立即用):**`Inter`**(把字距收紧、字重放轻即可逼近)或 `Helvetica Neue`。
- 中文:**`Noto Sans SC`**(与 grotesque 协调),回退 `PingFang SC`。
- 原则:**一套无衬线走天下**,不混入衬线;层级全靠字号/字重/留白。

> 当前框架已用 Inter,可先沿用;若你要更"Takram",我们再换授权字体。

## 4. 色彩 Color

近黑白灰,**单色优先**。

| Token | 值 | 用途 |
|---|---|---|
| `--paper` | `#FFFFFF` | 主背景 |
| `--paper-2` | `#FAFAF8` | 备用浅底(整段交替时用) |
| `--ink` | `#0E0E10` | 主文字 / 标题 |
| `--ink-2` | `#4A4A48` | 次级文字 |
| `--muted` | `#8A8A84` | 标签 / 信息 / 注释 |
| `--media` | `#F1F0EC` | 图片占位底(方正、无边框) |
| `--dark` | `#121214` | 深色大块(如 CMF 条 / 深色区) |
| `--hairline` | `rgba(14,14,16,.08)` | 仅极少数必要分隔 |
| `--accent` | *默认不启用* | 如需:仅用于链接/小标签,保持低调 |

要点:**默认不上彩色**。强调色留作以后(若品牌需要)再开,且只点缀链接或类目标签。

## 5. 图像 Imagery
- 满幅或大尺寸,**方角(0 圆角)、无边框、无阴影**。
- 占位用 `--media` 纯色平铺,居中极小标签即可,不要描边。
- 比例混用以制造节奏:`16:9`(主图)、`4:3`/`1:1`(辅图)、满幅大图穿插。
- 索引页支持 **hover 线框图→渲染图** 的交叉淡入(沿用之前思路),切换平滑、无边框。
- 视频/动图 tile 可自动播放、静音、循环(Takram 大量用短视频)。

## 6. 区块规范 Sections(都不使用框/线)

**导航 Nav** — 顶部极简:左 logo 字标,右 `Projects / About / Contact` + 语言(EN/中)。透明或纯白底,**无下边框**;滚动时可加极淡背景。

**Hero** — 一句大标题(Display)+ 一句引言(Lead),四周大留白。不放框、不放按钮框,链接用纯文字 + 下划线(hover 才显)。

**项目索引 Index(首页主角)** — 大图网格,桌面 2 列为主、可穿插满幅特写;移动 1 列。
每个项目:**大图(方角)→ 下方标题 + 类目小标签**,纯文字、无卡片框。区块之间留白拉开,不画线。

**项目详情 Detail** — 叙事仍用 `背景 / 概念 / 推进 / 成果`,但呈现去掉所有框线:
- 标题区:项目名(H3)+ 中文副名 + 引言 + 一行 meta(纯文字,**去掉虚线下划线**)。
- 主图:满幅/大幅方角图。
- 四段:小标签(Label)+ 段落(Body),用**留白和两栏节奏**区分,不加边框/底色。
- 辅图:2–3 张方角图,间距拉开。
- CMF:整条 `--dark` 深色块(方角、无边框)即可。
- 返回/下一项目:纯文字链接,无按钮框。

**页脚 Footer** — 极简多列文字(联系 / 媒体 / 社交),纯文字链接,无分割线,大留白收尾。

---

## 7. 动效 Motion
- 时长 300–600ms,缓动 `cubic-bezier(0.2, 0.7, 0.2, 1)`。
- 进入:轻微淡入 + 上移 16–24px。
- 图像 hover:线框→渲染交叉淡入,或极轻微放大(≤1.02)。
- 平滑滚动;避免花哨,**克制**为上。

---

## 8. 当前 Figma 框架要改的点(落地清单)
> 现框架是"有圆角+描边+分割线"的低保真版,按本规范需要:

1. **圆角全部 → 0**(图位、深色块等)。
2. **去掉所有占位描边**(图位改纯 `--media` 平铺,无边框)。
3. **去掉区块标题的底色块**(A/B 板块标题直接落在白底,用大字号+小标签区分)。
4. **去掉每个项目顶部的分割线**,改用更大的竖向留白拉开项目。
5. **去掉 meta 的虚线下划线**,改纯文字。
6. **字距收紧、标题字重放轻**,贴近 grotesque 编辑感。
7. CMF 深色条保留,但**方角、无边框**。
8. 整体**加大留白**(区块间距向 160–200px 靠拢)。
